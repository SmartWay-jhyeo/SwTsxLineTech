# 시공얼마(SmartWay) 플랫폼 전환 기획안 (2025-12-12)

## 1. 프로젝트 개요
*   **목표:** 기존 시공 업체 웹사이트를 **'투명한 견적 기반의 중개 플랫폼'**으로 전환
*   **핵심 가치(USP):** "3초 만에 확인하는 내 건물 시공 견적" (불투명한 시장 관행 타파)
*   **비즈니스 모델(BM):**
    *   **직영 시공:** 경기권 및 인근 지역 (마진 확보)
    *   **중개 매칭:** 원거리 및 일정 초과 건 -> 타 업체 연결 (건당 수수료/매칭비 5만원 수익)

## 2. 타겟 오디언스
1.  **주차장 도색:** 빌라 관리인, 상가 건물주, 신축 공사 현장 소장
2.  **바닥 공사(에폭시):** 공장장, 물류창고 관리자, 상가 인테리어
3.  **내/외벽 페인트:** 노후 건물 리모델링, 상가 입주민

## 3. 단계별 실행 계획 (Roadmap)

### [Phase 1] 랜딩 페이지 & UX 전면 개편 (The Hook)
*   **목표:** 방문자가 들어오자마자 이탈하지 않고 '견적'을 확인하게 함.
*   **변경 사항:**
    *   **메인 히어로(Hero) 섹션:** 단순 이미지 슬라이더 제거 -> **[간편 견적 계산기]** 전진 배치.
    *   **탭(Tab) 구조:** 에폭시 / 차선도색 / 페인트 3개 탭으로 구성하여 즉시 접근.
    *   **UX 흐름:** 평수/재질 선택 -> 대략적 견적 범위(Min-Max) 노출 -> "정확한 견적서 받기" 클릭 시 DB 수집(상담 신청).
    *   **신뢰 강화:** 견적 결과 하단에 관련 시공 사례(Portfolio) 자동 추천 노출.

### [Phase 2] 관리자(Admin) 시스템 고도화 (The System)
*   **목표:** 들어온 문의(Lead)를 효율적으로 분류하고 '돈'으로 만드는 관제탑 구축.
*   **변경 사항:**
    *   **DB 스키마:** 견적 요청 테이블에 `status` (접수/직영/중개/완료) 및 `memo` (배정 업체명, 특이사항) 컬럼 활용.
    *   **Admin UI:** 
        *   들어온 문의를 리스트로 확인.
        *   버튼 클릭으로 '직영' vs '중개' 상태 변경 기능.
        *   중개 시 배정된 업체 정보를 메모할 수 있는 기능.

### [Phase 3] SEO 및 콘텐츠 전략 (The Traffic)
*   **목표:** 돈 안 드는 오가닉 트래픽(Organic Traffic) 극대화.
*   **전략:**
    *   **Programmatic SEO:** 시공 사례 페이지의 Title/Description을 "지역명 + 시공종류 + 가격" 패턴으로 자동 생성.
        *   *Before:* "공장 에폭시 시공 사례 1"
        *   *After:* "수원 권선구 50평 공장 에폭시 바닥 시공 비용 공개"
    *   **구조화 데이터(JSON-LD):** 검색엔진이 '상품/서비스'와 '가격'을 인식하도록 마크업 강화.

### [Phase 4] 테스트 및 배포
*   **Branch:** `feature/platform-pivot` 에서 개발.
*   **Test:** `npm run dev` 및 로컬 빌드 테스트.
*   **Deployment:** Vercel 배포 및 운영 DB 마이그레이션.

## 4. 기술적 요구사항
*   **Frontend:** Next.js 14+ (App Router), Tailwind CSS
*   **Backend:** Supabase (PostgreSQL)
*   **Deployment:** Vercel
